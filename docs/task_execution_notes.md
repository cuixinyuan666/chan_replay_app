# 任务进行时注意事项

本文件记录 `vespa_tdx` 后续任务执行时必须反复检查的事项，避免重复出现“前端入口重复、删除不彻底、第三方图表坐标不同步、误改算法”等问题。

## 固定检查项

1. **先看用户修复后的 git diff，再动代码。**
   - 每次继续任务前先比较最近若干提交，确认用户已经修过哪些问题。
   - 不允许在不了解用户修复内容的情况下覆盖式重写。

2. **删除类需求必须全链路删除。**
   - 不只删 UI。
   - 必须同时检查：import、enum、source file、README、docs、搜索残留、构建产物引用。
   - 关键词必须搜索，例如 `TencentKlineSource`、`tencent`、`腾讯`、`market`、`count`。

3. **单入口原则必须逐页检查。**
   - 同一个设置项只能有一个入口。
   - 数据源、标的、周期、复权、日期只在数据源面板。
   - 一次性/逐K、图层、缩放只在左侧工具栏。
   - Vespa/chan.py 引擎参数只在设置面板。

4. **不可用操作必须灰度不可点击。**
   - 无数据时：缩放、最新、图层切换、逐K/一次性切换等应禁用或明确不可操作。
   - CSV 模式下：远程行情参数不可编辑。
   - 加载中：禁止重复加载和切换会产生并发状态的控件。

5. **引入第三方图表组件后必须检查坐标和手势。**
   - 不允许假设第三方图表绘图区与自绘叠加层一致。
   - 必须检查右侧价格轴、顶部/底部 padding、时间轴、手势冲突。
   - 当前 `candlesticks` 底层图使用 `IgnorePointer`，交互由外层统一接管；后续改动必须保持 K线、分型、笔、线段、中枢同一套可见窗口和坐标映射。

6. **前端问题不能通过改缠论算法解决。**
   - K线图错位、提示重叠、按钮入口重复，都属于前端问题。
   - 不允许为了让图形“看起来合理”修改 FX/BI/SEG/ZS 规则。

7. **缠论逻辑只允许对齐 Vespa/chan.py。**
   - 新增或修改算法前必须先做 Vespa 对齐测试。
   - 差异修复顺序：先 BI，再 SEG，再 ZS。
   - 不允许自造分型、笔、线段、中枢规则。
