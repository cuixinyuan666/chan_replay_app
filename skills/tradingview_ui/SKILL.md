# TradingView 风格 UI 与交互复刻 Skill

## 目标

本 skill 用于指导 `chan_replay_app` 的 UI 和操作逻辑设计，使缠论 K 线复盘 App 具备接近 TradingView 的图表工作台体验。

核心目标不是复制品牌标识，而是复刻交易图工作台的交互结构：顶部行情工具栏、左侧绘图工具栏、中央可拖拽缩放图表、右侧信息/观察面板、底部回放控制区。

## 布局原则

1. 顶部工具栏
   - 放置市场、代码、周期、复权、数据源、设置入口。
   - 常用周期要可快速切换，避免深层菜单。
   - 当前数据源状态应始终可见。

2. 左侧垂直工具栏
   - 放置光标、分型、笔、中枢、画线/标注等图表工具。
   - 图表显示开关应靠近图表，不放进深层设置。
   - 当前启用项使用高亮底色。

3. 中央图表区
   - 图表是主视觉，占据最大面积。
   - 保留右侧价格轴、底部时间轴、暗色网格。
   - K 线、分型、笔、中枢是图层关系：K线在底层，中枢区域在其上，笔线和分型在更上层。

4. 右侧信息面板
   - 显示当前品种、周期、数据源、K线数量、分型/笔/中枢统计。
   - 显示最后一根 K 线的 OHLCV。
   - 小屏幕下可隐藏，避免挤压图表。

5. 底部回放条
   - 保持播放、暂停、前进、后退、重置和进度拖动。
   - 回放控制区应与图表区分离，但视觉风格保持一致。

## 图表交互原则

1. 拖拽平移
   - 用户横向拖动图表时，视窗向历史或最新方向移动。
   - 视窗不能越过已有数据边界。

2. 双指缩放 / 滑块缩放
   - 双指放大时减少可见 K 线数量。
   - 双指缩小时增加可见 K 线数量。
   - 缩放范围必须受控，避免蜡烛过密或过稀。

3. 十字光标
   - 点击或长按图表时显示十字光标。
   - 十字光标应定位到最近 K 线。
   - 显示对应 K 线的时间、OHLCV 信息。

4. 回到最新
   - 当用户加载新数据、切换数据源或重置时，视窗回到最新位置。
   - 当用户手动平移后，保留其查看历史的视图，直到再次重置或加载数据。

5. 图层开关
   - 分型、笔、中枢必须可以快速开关。
   - 关闭图层只影响绘制，不影响底层缠论引擎计算。

## 视觉原则

1. 暗色工作台
   - 背景使用深色，网格使用低透明度线条。
   - 价格轴、时间轴、标签使用灰白文字。

2. 颜色约定
   - 上涨 K 线用红色。
   - 下跌 K 线用绿色。
   - 分型、笔、中枢使用相互区分的辅助色，但不压过 K 线。

3. 信息密度
   - 图表区尽量少放大按钮和卡片。
   - 状态信息放在顶部栏、右侧面板或十字光标浮层。

## 代码约束

1. 缠论计算链路不应因 UI 重构而改变。
2. `RawBar -> ChanSnapshot -> KlineChart` 的数据流保持稳定。
3. UI 状态，例如视窗位置、十字光标、缩放窗口，放在页面状态层或图表组件状态层。
4. 绘图组件只负责绘制和基础手势映射，不直接修改缠论引擎。
5. 所有新交互都必须在无数据状态下安全返回，不抛异常。
